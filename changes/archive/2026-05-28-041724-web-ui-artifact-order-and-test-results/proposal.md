# Proposal: web-ui-artifact-order-and-test-results

## Why

Web UIのチェンジ詳細画面でアーティファクトの表示順が不定であり、ワークフローの実行順に沿っていないため、ユーザーが現在の進捗を把握しにくい。また、テストリザルトの表示データがチェンジをまたいで共有される構造になっており、チェンジごとに独立した結果管理ができていない。さらに、テストリザルトとチェックリスト項目の紐づきがUIで確認できず、品質確認の文脈が失われている。

## Goals

1. チェンジ詳細画面のアーティファクト一覧を、mspecワークフローのステップ順（new → proposal → delta → research → design → quickstart → checklist → tasks → implement）に固定ソートする。
2. テストリザルト（red / green / skip）を、チェンジ固有のエージェントJSONフィールドとして出力する構造に変更し、Web UIはそのフィールドを参照してテストリザルト画面を構成する。
3. テストリザルト画面で、各テストケースが紐づくチェックリスト項目を表示する（n:1マッピング）。紐づきが壊れた場合は「未解決」として警告表示する。

## Non-Goals

- パフォーマンス最適化（表示速度の原声最適化は対象外）
- モバイル対応（レスポンシブデザインの改善は対象外）
- テストリザルトの自動実行（テスト実行自体の変更は行わない、表示のみ改善）
- 既存チェックリスト形式の変更（`checklist.md` のフォーマット自体は変更しない）

## Capabilities (touched)

- `change-dashboard` — アーティファクト一覧のソート順をワークフローステップ順に変更
- `test-result-viewer` — テストリザルトのデータソースをエージェントJSONフィールドに変更し、チェックリスト紐づきを表示
- `agent-runner` — エージェントJSONに `red` / `green` / `skip` テストリザルトフィールドを追加出力する仕様を追記

## Decisions

| 質問 | 決定 | 理由 |
|------|------|------|
| スタックトレース露出対策 | サーバー側でサニタイズ（絶対パスをマスク） | 開発者ツールだが内部パスの意図しない露出を防ぐ |
| XSS対策レイヤー | フレームワーク（React等）の自動エスケープに委譲（dangerouslySetInnerHTML不使用） | シンプルかつ標準的なアプローチ |
| Dangling reference処理 | テスト結果は表示し「チェックリスト項目：未解決」として警告表示 | データの可視性を保ちつつ問題を明示 |
| アクセス制御 | localhostのみ・認証不要 | ローカル開発専用ツールのため |
| シークレット混入防止 | ローカル専用環境のため追加対策不要 | ローカル開発環境のみでの利用前提 |

## Open Questions

- **エージェントJSONの実体**: `red`/`green` フィールドを追加する対象は `.agent-runs.jsonl`（既存）か、それとも新規の `test-results.json` ファイルか。現在 test-result-viewer は Playwright XML を解析する設計（FR-001）になっているが、今回の変更でデータソースをJSONに一本化するかどうかは research ステップで調査が必要。
- **チェックリスト紐づきのキー**: テストケース名とチェックリスト項目をどの識別子（チェックリスト行番号？FR番号？）で紐づけるか。
